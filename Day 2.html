Types of Keys in DBMS

Super Key

A super key is a set of one or more columns (attributes) that can uniquely identify a record within a table. It guarantees uniqueness for each row and can include more attributes than necessary.

Candidate Key

A candidate key is a minimal super key, meaning it is the smallest set of columns that can uniquely identify a record. No subset of a candidate key can uniquely identify a record on its own.

Primary Key

A primary key is a special type of key used to uniquely identify each record in a table. It ensures data integrity, efficient data retrieval, and the establishment of relationships between tables.

Alternate Key

An alternate key is a candidate key that is not chosen as the primary key. Alternate keys can still uniquely identify records in a table but serve as secondary unique identifiers.

Foreign Key

A foreign key is a field (or collection of fields) in one table that uniquely identifies a row of another table or the same table. It ensures referential integrity of the data.

Composite Key

A composite key is a key that consists of two or more columns (attributes) that together uniquely identify a record in a table. It is useful when a single column is not sufficient to uniquely identify rows.

Artificial Key

An artificial key, also known as a surrogate key, is a type of key used to uniquely identify each record in a table. Unlike natural keys, artificial keys are generated and have no inherent meaning outside of uniquely identifying a record.

Books Table

Genre Table

Genre ID Genre Name
1 Fiction

Conclusion from Example

  • Primary Key: Book ID
  • Foreign Key: Genre ID
  • Super Key: {Book ID}, {Book ID, Title}
  • Candidate Key: {Book ID}
  • Composite Key: {Book ID, Genre ID}
  • Artificial Key: Book ID
